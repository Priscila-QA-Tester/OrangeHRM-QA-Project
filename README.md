
# OrangeHRM QA Manual Testing Project

Welcome to my Quality Assurance (QA) portfolio project! 
This repository contains a structured approach to manual testing, demonstrating my ability to analyze requirements, design test cases, and document defects.

## 🎯 Objective
The goal of this project is to validate the core functionalities of the **OrangeHRM** Open Source system, focusing on user flows and edge cases. It serves as a showcase of my foundational QA skills, complementing my automated testing projects.

## 📂 Project Structure
- `test-cases.md`: Contains 10 detailed manual test cases covering key modules (Login, PIM, Leave, Admin, Time), including negative and edge cases.
- `gherkin.feature`: Shows my ability to translate manual scenarios into BDD (Behavior-Driven Development) format using Given-When-Then syntax.
- `bug-reports.md`: Standardized defect tracking template and a log of bugs found during exploratory testing.

## 🛠️ Tools & Methodologies
- **Techniques:** Exploratory Testing, Functional Testing, Boundary Value Analysis.
- **Documentation:** Markdown, Gherkin (Cucumber syntax).
- **Bug Tracking:** Jira Kanban Simulation.
- **Test Environment:** https://opensource-demo.orangehrmlive.com/
